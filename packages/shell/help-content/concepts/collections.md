# Collections

A **collection** is a named group of [entities](./entities.md). Use it for a project, a topic, an area of your life, a reading list, anything you want to keep together.

## How they work

A collection is itself an entity — it has a name, a description, an icon. What makes it a collection is membership: it points at the entities that belong to it.

An entity can live in many collections at once. A note about wine pairing can be in `Cooking`, `Travel: France`, and `Holiday-2026` simultaneously.

## Create one

From the dashboard launcher, type the collection name and pick **Create collection**. Or in any app, right-click an entity and choose **Add to collection** → **New collection**.

## Add and remove

- Drag an entity onto a collection in the sidebar.
- Right-click → **Add to collection**.
- Open the collection and use **Add** at the top.

Removing an entity from a collection doesn't delete the entity — it just leaves the collection.

## Collection views

Open a collection and you see its members. The default view is a grid; switch to list, gallery, board, calendar, or timeline using the view picker. Filter, sort, and group from the toolbar. The [Database app](../apps/database.md) is built on this — every Database view is a view over a collection.

## Smart collections

A smart collection has its members defined by a filter, not by hand. "All tasks due this week", "All notes tagged `idea`". As entities match or stop matching the filter, the collection updates automatically. Create one from **New collection → Smart**.
