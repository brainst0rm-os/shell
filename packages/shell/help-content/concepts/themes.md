# Themes

A **theme** controls how Brainstorm looks — colours, the typeface, icon style, accent. Change it from **Settings → Appearance**.

## Built-in themes

Brainstorm ships with a small set: **Light**, **Dark**, **System** (follows your OS), plus a few accent variants. Switching is instant — no restart.

## Installing more

The [Marketplace](../getting-started/install-an-app.md) has additional themes from us and from third-party designers. Install a theme like you'd install an app; it shows up in **Settings → Appearance → Theme**.

A theme can't change app behaviour or read your data — it only ships colours, fonts, and icons. So themes are safe to try.

## What's themable

- **Colours** — backgrounds, surfaces, text, accents, borders.
- **Typography** — body font, monospace font, font size scale.
- **Icons** — the icon pack used across the shell and apps.
- **Wallpapers** — the dashboard background.

Apps inherit the theme — they don't ship their own. So switching themes affects every app uniformly.

## Custom CSS

If a theme isn't quite right, **Settings → Appearance → Custom CSS** lets you write CSS overrides that apply on top of the active theme. This is for advanced users; bad CSS can hide buttons or break the layout. The editor warns you about obvious mistakes.

## Theme editor

Building a theme from scratch is coming. For now, the recommended path is: pick the closest built-in theme, then override what you don't like in custom CSS.
